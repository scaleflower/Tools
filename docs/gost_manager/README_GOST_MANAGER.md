# GOST 端口映射管理工具 使用说明

## 概述

这是一个交互式的GOST端口映射管理工具，支持管理多个GOST服务器的配置。

## 快速开始

```bash
cd /Users/heyake/Documents/Jacky/gost/scripts
python3 gost_manager.py
```

## 功能详解

### 1. 选择GOST服务器

切换当前操作的GOST服务器。显示格式：

```
┌────┬──────────┬─────────────────────────────────┐
│ 序号│  名称    │          API地址               │
├────┼──────────┼─────────────────────────────────┤
│  1  │ YTLC1    │ http://192.168.123.239:8090    │
│  2  │ YTLC2    │ http://192.168.123.224:8090    │
│  3  │ ECSHK    │ http://192.168.123.1:18080     │
│  4  │ FSMTC    │ http://192.168.123.180:8090    │
│  5  │ KTRN     │ http://192.168.123.163:18080   │
└────┴──────────┴─────────────────────────────────┘
```

### 2. 查看所有服务

列出当前服务器上所有已配置的端口映射服务。

### 3. 添加单个服务

添加一个新的端口映射服务。**操作前会自动备份当前配置。**

**输入项：**
- 服务名称：唯一标识符，如 `ktrn-pr-ocs01-ssh`
- GOST监听端口：GOST服务器监听的端口，如 `52001`
- 目标IP地址：内网目标服务器IP，如 `192.168.20.140`
- 目标端口：目标服务端口，如 `22`

### 4. 删除单个服务

根据服务名称删除指定的端口映射。

### 5. 批量添加服务

批量添加多个端口映射服务。**操作前会自动备份当前配置。**

**输入格式（用空格分隔）：**
```
服务名称    GOST端口    目标IP    目标端口
```

**示例：**
```
ktrn-pr-ocs01-ssh 52001 192.168.20.140 22
ktrn-pr-app01-ssh 52011 192.168.20.152 22
my-db-forward     53000 192.168.10.100 3306
```

输入空行结束输入，然后确认批量添加。

### 6. 删除所有服务

删除当前服务器上的所有端口映射服务。

**⚠️ 危险操作，需要验证：**
1. 输入管理密码：`iWhale@2023`
2. 输入确认文本：`DELETE ALL`

### 7. 测试服务器连接

测试当前服务器的API连接是否正常。

### 8. 备份服务器配置

手动备份当前服务器的完整配置到本地。

### 9. 从备份恢复配置

从本地备份文件恢复配置到当前服务器。

**恢复流程：**

1. **选择备份文件**
   ```
`
   可用备份文件:
   --------------------------------------------------
    1. KTRN_20260403_150530.json    2026-04-03 15:05:30 (42KB)
    2. KTRN_20260403_144439.json    2026-04-03 14:44:39 (38KB)
    3. YTLC1_20260403_144359.json   2026-04-03 14:43:59 (35KB)
   --------------------------------------------------
    0. 返回
   ```

2. **自动备份当前配置**
   - 恢复操作前会自动备份当前配置（防止误操作）

3. **选择恢复模式**
   - `y` = **清空后恢复** - 先删除所有现有服务，再恢复备份
   - `n` = **追加恢复** - 保留现有服务，添加备份中的服务
   - `0` = 取消操作

4. **执行恢复**
   - 显示恢复进度
   - 统计成功/失败数量

**恢复模式选择建议：**
| 场景 | 推荐模式 |
|------|---------|
| 配置丢失/损坏 | 清空后恢复 (y) |
| 追加新服务 | 追加恢复 (n) |
| 不确定当前状态 | 清空后恢复 (y) |

## 备份说明

### 自动备份

以下操作会自动触发备份：
- 添加单个服务
- 批量添加服务

### 手动备份

通过菜单选项 `8. 备份服务器配置` 手动执行备份。

### 备份文件位置

```
/Users/heyake/Documents/Jacky/gost/scripts/backups/
```

### 备份文件格式

```
{服务器名}_{日期时间}.json
```

**示例：**
```
KTRN_20260403_150530.json
YTLC1_20260403_151000.json
```

### 备份文件内容

```json
{
  "backup_info": {
    "server_name": "KTRN",
    "server_url": "http://192.168.123.163:18080",
    "backup_time": "2026-04-03 15:05:30",
    "backup_by": "gost_manager"
  },
  "config": {
    "services": [...],
    "authers": [...],
    "chains": [...],
    ...
  }
}
```

## 服务配置格式

每个端口映射服务的配置格式：

```json
{
    "name": "ktrn-pr-ocs01-ssh",
    "addr": ":52001",
    "handler": {
        "type": "tcp"
    },
    "listener": {
        "type": "tcp"
    },
    "forwarder": {
        "nodes": [
            {
                "name": "ktrn-pr-ocs01",
                "addr": "192.168.20.140:22"
            }
        ]
    }
}
```

## 安全说明

1. **鉴权信息已内置**：API认证信息已配置在脚本中，无需手动输入
2. **危险操作保护**：删除所有服务需要输入管理密码
3. **自动备份**：添加服务前自动备份，可回滚

## 常见使用场景

### 场景1：批量添加新服务器端口映射

1. 选择GOST服务器
2. 批量添加服务
3. 按格式输入多个映射
4. 确认添加

### 场景2：误删恢复

**使用恢复功能：**
1. 选择GOST服务器
2. 选择 `9. 从备份恢复配置`
3. 选择对应的备份文件
4. 选择恢复模式（建议选择清空后恢复）
5. 确认恢复

### 场景3：迁移配置到新服务器

1. 选择源服务器
2. 选择 `8. 备份服务器配置`
3. 切换到目标服务器
4. 选择 `9. 从备份恢复配置`
5. 选择刚才的备份文件
6. 选择恢复模式

### 场景4：灾难恢复

如果GOST服务器配置完全丢失：

1. 重新部署GOST服务
2. 运行管理工具
3. 选择该服务器
4. 选择 `9. 从备份恢复配置`
5. 选择最近的备份文件
6. 选择 `y` 清空后恢复

## 注意事项

1. 首次使用请先选择服务器
2. 批量添加前建议先手动备份一次
3. 删除操作不可恢复，请谨慎操作
4. 备份文件包含完整配置，可用于灾难恢复
